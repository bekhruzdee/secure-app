🛡️ Security Guide

Select a threat type for details

Cross-Site Scripting (XSS)

What it does: Attackers inject malicious scripts into web pages to steal cookies, tokens, or perform actions on behalf of users.

Detection: We monitor for HTML/JS patterns like <script>, event handlers (onload=), and javascript: URIs.

Examples blocked:

  • <script>alert('hacked')</script>
  • <img src=x onerror=fetch('/steal')>
  • <iframe src="javascript:alert('xss')">

Cross-Site Request Forgery (CSRF)

What it does: Attackers trick authenticated users into making unwanted requests (transfers, deletions) without their knowledge.

Detection: We require a valid CSRF token on POST/DELETE requests. Missing or invalid tokens are blocked.

How we protect:

  • Generate unique tokens per session
  • Store tokens in secure cookies
  • Validate tokens on state-changing requests
  • Login endpoint is exempt (public access)

SQL Injection (SQLi)

What it does: Attackers inject SQL code into input fields to bypass authentication, steal data, or modify databases.

Detection: We scan for SQL keywords and patterns like UNION SELECT, DROP TABLE, OR 1=1.

Examples blocked:

  • ' OR '1'='1
  • admin'; DROP TABLE users; --
  • 1 UNION SELECT * FROM passwords
